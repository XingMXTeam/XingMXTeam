# Pinned repository About settings

GitHub pinned cards rely heavily on repository metadata: description, website, and topics. README quality is not enough.

This file records the recommended About settings for repositories that may be pinned on the public GitHub profile.

## Current pin strategy

Do not pin weak or forked repositories just to fill six slots.

Recommended current public pins:

1. `xiaohongshushipfast`
2. `a-simple-mcp`
3. `crawler_ai_chrome`
4. `frontend-map`

Optional only if public and useful:

5. `maoxunxing.com` — only if the repository is public; otherwise use the profile README link to the site instead of pinning it.

Do not pin:

- `rag-search` — forked / reference-based learning experiment.
- `agent-workstation` — private until public-ready.
- `ebook-maker` — private until a clean public template is extracted.

## Manual steps

For each repository:

1. Open the repository page.
2. Find the right-side **About** panel.
3. Click the gear icon.
4. Fill **Description**.
5. Fill **Website** when useful.
6. Add **Topics**.
7. Save changes.

Then go to the GitHub profile page and update pinned repositories manually.
